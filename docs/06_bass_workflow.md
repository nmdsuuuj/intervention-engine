# 06. ベース・ワークフロー仕様書

## 1. 概要 (Overview)
ベースを「弾けない」ユーザーでも「豊かな表現」を可能にするための、「生成」と「構造編集」のワークフローを定義する。

## 2. 高精度な「パターン生成」
* **`[ コード生成 ]` 時:** `[Bass]` トラックには、単なるルート音ではなく、`assets/data/bass_patterns.json` に格納された高精度なパターン（例：`pattern_id: "funk_syncopation_01"`, `pattern_id: "mccartney_walking_01"`）が自動で適用される。
* **`[ MUTATE ]` 時:** ユーザーは「奏法（アーティキュレーション）」の介入も提案される。
    * **例：** `[ 提案A: スライド・アップ ]` (`art_slide_up_fast` を適用), `[ 提案B: ポール風フィル ]` (`art_mccartney_fill_01` を適用)

## 3. 構造編集（コードへの追従）
* **目的:** 1音ずつ編集するのではなく、「コードとベースの関係性」を編集する。
* **ワークフロー:**
    1.  ユーザーが `[Chord]` トラックで、コードを `Am` から `A7` に変更する。
    2.  「介入エンジン」が、`[Bass]` トラックのノート（`C`=ド）との「衝突」を**自動で検知**する。
    3.  **ポップアップが表示される。**

> **[ポップアップ]**
> 「`[Chord]` が変更されました。`[Bass]` トラックの衝突をどうしますか？」
>
> 1.  **[ 追従させる (Conform) ]**
>     * → ベースの「ド」を、自動で「ド#」に修正します。（最も簡単な編集）
>
> 2.  **[ 介入する (MUTATE) ]**
>     * → 新しいコード `A7` に対する「複数のベースパターン（`docs/02`の提案）」を今すぐ表示します。
>
> 3.  **[ 無視する (Ignore) ]**
>     * → この「衝突（奇妙さ）」を意図的に残します。（ビートルズ的）
