# 09. 将来の拡張機能 仕様書

## 1. 概要 (Overview)
v1.0（初期リリース）のコア機能（介入エンジン＋MIDIシーケンサー＋Audio 1トラック）の完成後、アプリを「真の制作ツール」に進化させるための、優先度の高い拡張機能ロードマップを定義する。

## 2. v2.0：4トラックMTR（マルチ・トラック・オーディオ）機能 (最優先)

* **要求:** ユーザー（あなた）からの「4track mtr機能を拡張機能の最初の方に」という要求に基づく。
* **目的:** 現在の「1つのオーディオ（歌）レーン」を、**「4つの独立したオーディオ録音レーン」**に拡張する。
* **仕様:**
    1.  `docs/01` の「アレンジ・ビュー」の `[Audio (歌)]` レーンを、`[Audio 1]`, `[Audio 2]`, `[Audio 3]`, `[Audio 4]` の4レーンに拡張する。
    2.  ユーザーは、MIDIトラックを再生しながら、**オーディオを4トラック分、個別に重ね録り（オーバーダビング）**できるようになる。
    * **例:** `[Audio 1]`: メインボーカル, `[Audio 2]`: ギター, `[Audio 3]`: ベース, `[Audio 4]`: コーラス
* **影響:** これにより、このアプリは「MIDIスケッチパッド」と「オーディオMTR」のハイブリッドとなり、DAWに近い柔軟な制作が可能になる。

## 3. v3.0：自動文字起こし (Speech-to-Text)
* **要求:** ユーザー（あなた）からの「自動文字起こし出来たら楽」という要求に基づく。
* **目的:** `[Audio]` レーンのボーカル録音を解析し、`[Chord / Lyric]` レーンに歌詞を自動で入力する。
* **技術:** 「音声認識AI（Speech-to-Text）」という、介入エンジンとは別のAI技術を必要とする。

## 4. v4.0：ドラムの鼻歌（ボイスパーカッション）入力
* **要求:** ユーザー（あなた）からの「リズムやベースを鼻歌で」という要求に基づく。
* **目的:** 「ドン」「チッ」「タッ」という音色を分類し、ドラムパターンを自動生成する。
* **技術:** 高度な音響分類（ML）モデルを必要とする。
